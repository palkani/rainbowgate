# Product & category images

The `.svg` files here are **branded placeholders** generated by
`scripts/gen-placeholders.mjs`. Replace them with real photography before
launch — the site will pick up the new files automatically as long as you
follow the naming convention below and update the `image` path in
[`lib/products.ts`](../../lib/products.ts).

## Naming convention

| Use            | Aspect ratio | Recommended size | File name pattern              |
| -------------- | ------------ | ---------------- | ------------------------------ |
| Product photo  | 1:1 (square) | 1000 × 1000 px+  | `<product-slug>.jpg`           |
| Category tile  | 3:4 (portrait) | 900 × 1200 px+ | `category-<category-slug>.jpg` |

- The **slug** is the `slug` field of each product in `lib/products.ts`
  (e.g. `bamboo-water-bottle`). Category slugs are the `slug` of each entry in
  the `categories` array (e.g. `drinkware`).
- Prefer `.jpg` or `.webp` for photos. Keep each file under ~300 KB —
  `next/image` will still re-encode and resize, but smaller sources are faster.
- Shoot on a clean, neutral background for a premium, consistent grid.

## How to swap a placeholder

1. Drop your photo in this folder, e.g. `public/products/bamboo-water-bottle.jpg`.
2. In `lib/products.ts`, change that product's `image` from
   `/products/bamboo-water-bottle.svg` to `/products/bamboo-water-bottle.jpg`.
3. Update `imageAlt` to describe the actual photo (never leave it generic).
4. Once **all** placeholders are real raster images, you can remove the
   `dangerouslyAllowSVG` block from `next.config.mjs`.
